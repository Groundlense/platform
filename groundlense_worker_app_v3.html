1A

Login + Create Account

First screen the worker sees when opening the app. Split into two tabs — Login for existing workers, Create Account for new joiners. Designed to be completed in under 2 minutes on a phone in field conditions.

Login Tab
  • Mobile number or GL-W-XXXX worker ID
  • 4-digit PIN — faster than password on site
  • Works completely offline
  • Syncs automatically when signal returns
  • Forgot PIN — contact supervisor note
Create Account — 3 Steps
  • Step 1: Name + Mobile + Education (picture tiles)
  • Step 2: OTP verification via SMS
  • Step 3: Set 4-digit PIN + confirm
  • System generates GL-W-XXXX unique ID
  • Worker shares ID with engineer for project assignment
Education Selector
  • 6 picture tiles — no typing needed
  • 10th / 12th / ITI / Diploma / B.E. / Other
  • All labels in Hindi and English
  • Tap to select — visual, not dropdown
  • Critical for people data compliance
Key Design Rules
  • Only 3 fields in Create Account — keeps abandonment low
  • Engineer adds Aadhaar + docs from web portal
  • PIN not password — faster for dirty hands + sunlight
  • Two separate screens — login does not mix with registration
Design Decision
Create Account collects only 3 fields on purpose. Complex onboarding gets abandoned on site. The engineer adds verification documents and additional details from the web portal after the worker registers. Worker gets access in 2 minutes.

Opens immediately after successful login. Worker can search for a project by its unique GL-PRJ ID (given by supervisor) or select from their assigned project list. Previous completed projects are shown below for reference.

Search by Project ID

  • Instant result — project name + assignment status
  • Not assigned = red error message in Hindi
  • Not found = amber warning shown
  • Solves team handover problem completely
Assigned Project List
  • Auto-populates from engineer assignment
  • Active projects shown at top
  • Previous completed shown below
  • Multiple active projects supported
  • Team assignment shown as chip
Edge Cases Handled
  • Project found but not assigned — red error
  • Project ID not found — amber warning
  • No projects assigned yet — empty state with instruction
  • Multiple active projects today — both shown
Project ID Format

Boring List

Shows all boreholes assigned to this worker on the selected project. Each boring has a unique GL-BH-XXXX-XX identifier. Four color-coded states make it immediately clear what needs to be done today.

Four Boring States
  • Green — Complete: All entries done, locked, report ready
  • Amber — Terminated: Stopped mid-boring, must resume today
  • Rust — Active: Currently in progress
  • Grey — Pending: Not started yet
Unique ID System
  • GL-BH-0047-03 = GL + BH + project + sequence
  • Links boring to project permanently
  • Solves multi-team handover tracking
  • Used in all reports and certificates
  • If team changes — ID stays the same
Project Header
  • Project ID shown in phone bar
  • Project name + team shown in chip
  • Change button to switch project
  • Worker always knows which project they are in
Progress Bar
  • Visual completion % for the day
  • Count of each state shown below
  • CTA button changes contextually
  • Resume for terminated, Start for pending
Design Decision
The GL-BH unique ID is more important than it appears. When Team A terminates BH-03 and Team B resumes it the next day, the same ID links both sessions. The IE firm sees a complete unbroken audit trail showing exactly who did what, when, and at what depth — even across different teams.

Rig Setup ★ New screen

Added from Apensia GT report gap analysis. Every real IS 1892 borelog header requires: rig type, borehole diameter, drilling fluid, and hammer type. Without this, Ground Lense's PDF output would look incomplete to any IE firm engineer in the first 5 seconds of review. This screen is completed once per boring, locked after the first SPT entry.

Why This Screen Exists
  • IS 1892 borelog header is mandatory in every GT report
  • Real Apensia report shows: Rotary Drilling · 150mm · Water · IS 6926
  • Without it, Ground Lense PDF is immediately rejected by IE firms
  • One-time setup per boring — not repeated every interval
  • Appears automatically on borelog sheet Annexure II
Fields Captured
  • Rig type: Rotary / Wash boring / Auger / Percussion
  • Borehole diameter: 100mm / 150mm / 200mm
  • Drilling fluid: Water / Bentonite mud
  • Hammer type: Auto-trip / Rope and pulley
  • Driller GL-D-XXXX ID (auto-linked from assignment)
Lock After First SPT
  • Same logic as Engineer Portal Setup section
  • Cannot be changed after boring starts
  • Prevents retroactive alteration of rig records
  • If wrong — engineer must unlock with IS clause reason
Auto-Populated Fields
  • Driller ID pulled from project assignment
  • Date and time auto-recorded
  • Boring ID (GL-BH) pre-filled
  • All selections stored in borelog header permanently
Gap fix — from real GT report analysis
This screen was identified as a critical gap by cross-referencing the Apensia Consultancy GT report for NH-748, Goa. Every borelog sheet header (Annexure II) shows rig type and drilling method. Ground Lense could not generate a complete IS 1892 borelog without this data. Screen 2B fills this gap permanently.

Navigate + Start Boring

Worker navigates to the planned boring location using Google Maps integration. GPS coordinates are captured automatically and stamped directly on the rig photo. Any deviation from the planned location greater than 3m is flagged automatically — the worker cannot change this.

Google Maps Navigation
  • Planned boring location pre-pinned by engineer
  • Live distance indicator — "45m away · Walk north"
  • Open Google Maps button for turn-by-turn
  • Map preview shows planned vs current location
GPS Auto-Capture
  • Records Easting, Northing, RL automatically
  • Calculates deviation from planned location
  • Deviation >3m = amber flag raised
  • Worker cannot change GPS — it is evidence
  • Flag appears on contractor portal cross-section
GPS Stamped Photo
  • Coordinates printed ON the photo itself
  • Not just in metadata — visually on image
  • Timestamp + boring ID + E + N + RL + deviation
  • Legal proof that boring was at that location
  • Cannot be faked after the fact
Weather Selection
  • 4 picture options — tap one
  • Clear / Cloudy / Rainy / Hot
  • No typing needed
  • Appears on boring log report
  • IS 2131 requires weather data
Design Decision
GPS deviation flags but does NOT block. The worker can still start the boring even if they are 10m off. This is intentional — preventing the contractor from claiming "the app stopped our work." The deviation is recorded as permanent evidence for the IE firm to act on.

SPT Entry — The Core Loop

The most important screen in the app. Worker enters blow counts for the current 1.5m depth interval. The app automatically calculates the raw N value and corrected N value using IS 2131 formulas and shows the soil interpretation in plain language. After photo and submission, the app loops back to this screen for the next depth.

Current Interval Only
  • Shows one interval at a time — not all three
  • Previous intervals shown as small green ticks below
  • Current interval highlighted in rust
  • Reduces confusion for ITI-pass workers
  • Worker cannot accidentally edit previous entries
Auto-Calculations
  • Raw N = sum of 15-30cm + 30-45cm blows
  • Corrected N = overburden correction (IS 2131)
  • Dilatancy correction applied if applicable
  • IS code table lookup → plain language result
  • "Medium dense sand" shown automatically
Loop Indicator
  • "Interval 3 of 13 planned" always visible
  • Dot progress row shows completion
  • Green = done, Rust = current, Grey = pending
  • Amber dot = terminated interval
  • Worker always knows where they are
Refusal + Dilatancy — New
  • Refusal button — triggers N=100 + asks mm penetrated before refusal
  • Real reports show "R" at 6.0m, 7.5m, 9.0m — partial depth matters
  • Dilatancy flag — "Fine sand below WT?" shown after N value
  • If yes → N'' = 15 + (N' − 15)/2 correction applied
  • IS 2131 requirement — previously missing from design
Design Decision — The Boring Loop
After Screen 6 (Sample Collection), the app automatically returns to Screen 4 for the next depth interval. The worker never manually navigates between screens during a boring. The loop continues: Screen 4 → 5 → 6 → 4 → 5 → 6 until boring is complete, terminated, or rock is encountered.

Soil Type Tiles
  • 4 main types with emoji photos
  • Sand / Clay / Silt / Rock
  • Hindi + English on every tile
  • Tap to select — visual, not text dropdown
  • Maps to IS 1498 classification automatically
Colour Swatches
  • 5 colour swatches to tap
  • Brown / Dark brown / Grey / Light / Blue-grey
  • No colour name typing needed
  • Worker matches visual to sample
  • IS code colour classification auto-applied
    • Voice Note in Hindi
    • Optional additional observation
    • Google Speech-to-Text API in Hindi
    • Worker speaks — app transcribes
    • Appears as note on boring log
    • Critical for unusual soil conditions
    Consistency Options
    • Loose / Medium / Dense / Stiff

    • Selected option highlighted in amber
    • Maps to IS 1498 descriptors

    Zero free text fields on this screen. An ITI-pass worker standing in sunlight with soil on their hands cannot type IS 1498 soil descriptions. They tap a picture.     Sample Collection

    Step 3 of the boring loop. Worker selects sample type, gets an auto-generated unique ID shown in large text, writes it on the tube, takes a photo, and confirms sealing. A 14-day countdown timer starts automatically. After submission, the app loops back to Screen 4 for the next depth.

    Auto-Generated Sample ID
    • GL-BH03-4.5-S1 format
    • GL = Groundlense
    • BH03 = boring number
    • 4.5 = depth in metres
    • S1 = first sample at this depth
    • Shown in LARGE text — worker writes on tube with marker
    14-Day Countdown Timer
    • Starts automatically on sample creation
    • Shows deadline date clearly
    • IE dashboard alerts when approaching
    • Report flags if lab results not entered by day 14
    • IS 2132 requirement for UDS samples
    Sample Types
    • S prefix = Disturbed SPT sample
    • U prefix = UDS thin wall tube
    • Each type has different ID prefix
    • Lab module links results to ID automatically
    • Chain of custody complete from field to lab
    After Submit
    • App loops back to Screen 4 automatically
    • Next depth interval ready for entry
    • Worker never navigates manually
    • Loop continues until boring ends
    Design Decision
    Handwritten sample ID is more reliable than QR scanning in field conditions — hands are dirty, sun is bright, and there is no printer on site. The large display of the ID ensures the worker writes it correctly. The lab technician types the same ID when entering test results, completing the chain of custody.

    • One tap to record, boring continues immediately
    What Is Captured
    • Water depth in metres (worker enters)
    • Time auto-recorded and locked
    • Date auto-recorded
    • Linked to boring ID automatically
    • IS 6935 compliant observation
    24hr Reminder System
    • Set automatically when WT is recorded
    • Push notification at exact same time next day

    • Brings worker back to Screen 11 popup

    • Both readings appear on boring log
    • Initial reading + 24hr stable reading
    • Rise/fall calculated automatically

    Rock Coring Mode

    A separate mode triggered when rock is encountered during boring. The SPT loop exits and rock coring begins. Worker enters run length and recovered length — core recovery % and RQD are calculated automatically. No formula knowledge required.

    Core Recovery Auto-Calc
    • Worker enters: run length + recovered length
    • App calculates: Core Recovery % = (recovered ÷ run) × 100
    • Classification shown: Excellent/Good/Fair/Poor
    • Per IS guidelines for rock quality
    RQD Auto-Calculation
    • Worker physically counts pieces ≥ 100mm
    • Types the count — app calculates RQD %
    • RQD = (length of pieces ≥100mm ÷ run) × 100
    • Classification: Excellent/Good/Fair/Poor/Very Poor
    • No formula knowledge needed from worker
    Weathering Grade
    • 4 picture tile options
    • Fresh / Slightly / Moderately / Highly
    • Mapped to IRC 78 rock classification
    • Appears on foundation report automatically
    Core Box Photo
    • Mandatory per IS 4078 storage code
    • GPS stamped like all other photos
    • Linked to run number and depth
    • Visible on IE dashboard immediately
    Design Decision
    Dark header bar distinguishes rock coring mode visually from SPT mode. The worker immediately knows they are in a different workflow. All calculations happen automatically — worker counts pieces physically and types one number. This is the only input requiring physical counting, which is appropriate because pieces must be physically measured on site.

    UDS Thin Wall Sampling

    Used when an undisturbed soil sample is required for consolidation, triaxial, or UCS tests. Different blue color scheme distinguishes it from disturbed SPT samples. The U prefix in the sample ID distinguishes it from S (SPT) samples throughout the system.

    UDS Sample ID Format
    • GL-BH03-9.0-U1 format
    • U prefix = undisturbed (vs S for SPT)
    • Write on BOTH the tube AND the core box
    • Lab links test results to this ID
    • Chain of custody complete from field to report
    Mandatory Confirmations
    • Both ends wax sealed — cannot be skipped
    • Stored upright — IS 2132 requirement
    • Both checkboxes must be ticked before Next
    • Failure to seal = compromised test results
    • These are legally required IS code steps
    Recovery Calculation
    • Worker enters tube penetration depth
    • Worker enters length recovered
    • Recovery ratio auto-calculated
    • Low recovery = flag to engineer
    Color Scheme
    • Blue header bar — not rust
    • Blue ID box — not green
    • Worker learns: Blue = undisturbed, Rust = SPT
    • No label reading needed — color = type
    Design Decision
    Blue color scheme for UDS is deliberate. The worker learns to associate rust/orange with SPT and blue with undisturbed samples. When they see a blue screen they immediately know this sample requires more careful handling. Color communicates faster than text on a construction site.

    Terminate Boring

    Triggered by the Terminate button on Screen 4. Termination is NOT the same as closing a boring. Terminate = pause with reason. Close (Screen 10) = permanent completion. The terminated boring shows as amber in the boring list and can be resumed by any assigned worker.

    6 Reason Tiles
    • End of day — most common on site
    • Equipment breakdown
    • Site obstruction
    • Engineer instruction
    • Weather — will resume
    • Other — with optional voice note
    Resume vs Permanent Stop
    • Yes, resume later → boring stays amber in list
    • No, permanent stop → goes directly to Screen 10 closure
    • Resume = different team can pick it up next day
    • Permanent = boring log closed and locked
    Auto-Recorded Data
    • Termination depth — exact metres
    • Timestamp — date and time
    • Worker GL-W-XXXX ID
    • Reason selected
    • Device IMEI at termination
    Handover System
    • Terminated boring visible to all assigned workers
    • New team arrives — taps Resume BH-XX
    • Screen 4 shows amber resume banner
    • Previous entries visible as read-only
    • New session linked to same GL-BH ID
    Design Decision
    Terminate ≠ Close. This distinction solves a real field problem — on NHAI projects borings are frequently paused at end of day and resumed next morning, often by different workers. The amber state in the boring list is a clear visual signal: this boring needs to be resumed today. Green = done, amber = needs attention.

    Resumed Boring

    When a worker taps Resume on a terminated boring, Screen 4 opens with a special amber banner showing the previous session details. The boring continues from where it stopped. Both sessions are linked under the same GL-BH ID for a complete audit trail.

    Resume Banner
    • Shows termination depth + time
    • Shows reason for termination
    • Shows previous worker name + GL-W ID
    • Shows new starting depth for Session 2
    • Cannot be dismissed — always visible
    Session Tracking
    • Loop indicator shows amber dot for terminated interval
    • Depth bar shows "Session 2 · new date"
    • Previous entries visible as read-only in summary
    • New entries from resumption depth onward
    • Both sessions linked under same GL-BH ID
    Audit Trail
    • Session 1 data permanently linked to Worker A
    • Session 2 data permanently linked to Worker B
    • IE firm sees complete handover history
    • Timestamps show gap between sessions
    • Legally defensible complete record
    Auto-Detection
    • App detects last entered depth automatically
    • New session starts from next interval
    • No manual configuration needed

    Device Binding — The Hidden Fraud Prevention Feature
    When a worker logs in on a phone for the first time, the device IMEI is recorded and linked to their GL-W ID. If the same login is used from a different device without authorization, the system flags it immediately. This prevents the most common fraud scenario on Indian infrastructure sites — an office employee entering fabricated field data remotely while claiming to be on site. Every entry must physically originate from the registered field device.
